## Requisitos Não Funcionais

| ID | Requisito | Descrição |
|-----|------------|------------|
| RNF01 | Usabilidade | O sistema deve possuir uma interface intuitiva e de fácil utilização para os funcionários do RU. |
| RNF02 | Desempenho | O sistema deve processar consultas e cálculos de insumos em tempo adequado. |
| RNF03 | Disponibilidade | O sistema deve estar disponível durante o horário de funcionamento do Restaurante Universitário. |
| RNF04 | Confiabilidade | O sistema deve minimizar erros no cálculo de insumos e na gestão do estoque. |
| RNF05 | Compatibilidade | O sistema deve funcionar corretamente em navegadores modernos. |
| RNF06 | Escalabilidade | O sistema deve suportar aumento no volume de dados e refeições sem perda significativa de desempenho. |
| RNF07 | Manutenibilidade | O sistema deve permitir futuras atualizações e correções com facilidade. |
| RNF08 | Atualização de Dados | O sistema deve refletir alterações no estoque em tempo próximo ao real. |
| RNF09 | Acessibilidade | O sistema deve seguir práticas básicas de acessibilidade para facilitar o uso por diferentes usuários. |
